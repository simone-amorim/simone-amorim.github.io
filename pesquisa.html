Arquitetura de Ecossistemas de Software

Arquiteturas de Ecossistemas surgiram como uma nova área de pesquisa com promissoras vantagens competitivas. Com base nessas vantagens, muitas empresas estão abrindo suas fronteiras arquiteturais para converter suas plataformas de software em um ecossistema. Este processo de migração tem sido feito, sem modelos específicos ou padrões conhecidos. Além disso, esta evolução é muitas vezes impulsionada pelo ambiente de negócios, que obriga a organização a mudar seus processos de negócio, e, transitivamente, o sistema de informação que os suporta. A rápida evolução tecnológica também induz uma forte pressão para modificar o sistema de informação, a fim de torná-lo apto para suportar novos requisitos que a plataforma tecnológica legada não foi capaz de cumprir. Minha pesquisa tem como objetivo estudar os aspectos relevantes do processo de evolução Arquiteturas de Ecossistemas. O objetivo final é desenvolver uma abordagem para melhorar o projeto destas arquiteturas. A abordagem tem a intenção de fornecer opções para determinar quais as decisões de projeto podem ser adequadas para os mecanismos de implementação, e como estes devem ser empregados para melhorar os atributos de qualidade das arquiteturas de ecossistemas.
